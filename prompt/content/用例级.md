请基于六边形架构，为 Quadra Content 服务实现一个完整的用例：

【用例名称】
PublishMovement（发布图文动态）

【业务要求】
- 文本内容或媒体资源至少一项非空
- medias 为 JSON 数组，包含 url、type、width、height
- 创建 Movement 聚合根
- 发布 MovementPublishedEvent（写入 outbox_event）
- 扇出写入 MovementInbox（调用 social 关注关系获取粉丝列表）

【技术要求】
- JDK 21
- Spring Boot 3
- MyBatis-Plus
- MySQL 8

【必须输出以下内容】
1. quadra-content-domain 模块：
   - Movement 聚合根（包含 publish 工厂方法，内部校验内容非空、medias JSON 列表格式）
   - Media 值对象（url、type、width、height）
   - MovementPublishedEvent

2. quadra-content-application 模块：
   - PublishMovementCommand
   - PublishMovementUseCase 接口
   - ContentApplicationService 实现（编排发布与扇出逻辑）
   - port.out：MovementRepositoryPort, TimelineRepositoryPort, EventPublisherPort

3. quadra-content-adapter-out 模块：
   - MovementRepositoryImpl（medias JSON 字段双向转换：Domain List ↔ JSON string）
   - MovementInboxRepositoryImpl（批量扇出写入粉丝收件箱）
   - OutboxEventRepositoryImpl（写事件表，与业务同事务）

4. quadra-content-adapter-in-web 模块：
   - ContentController（POST /api/v1/content/movements）

【强约束】
- 不允许跨层调用
- 不允许出现 DAO 直连 Controller
- Repository 必须操作聚合根
- Domain 层不能出现 Spring 注解

请严格按照分层结构输出代码。
