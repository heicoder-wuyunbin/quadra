请基于六边形架构，为 Quadra Recommend 服务实现一个完整的用例：

【用例名称】
RecordAction（记录用户行为）

【业务要求】
- 记录用户对目标的行为（VIEW/LIKE/SKIP/DISLIKE）
- 行为类型必须合法，非法类型抛出领域异常
- 根据行为类型计算权重分数
- 支持批量写入行为日志

【技术要求】
- JDK 21
- Spring Boot 3
- MyBatis-Plus
- MySQL 8

【必须输出以下内容】
1. quadra-recommend-domain 模块：
   - UserActionLog 聚合根（包含 record 工厂方法，内部必须校验行为类型合法 VIEW/LIKE/SKIP/DISLIKE）
   - ActionType 枚举（定义合法行为类型与权重）
   - UserFeature / ContentFeature / RecommendUser / RecommendContent 聚合根

2. quadra-recommend-application 模块：
   - RecordActionCommand
   - RecordActionUseCase 接口
   - RecommendApplicationService 实现
   - BuildUserFeatureUseCase、GetRecommendUsersQuery
   - port.out：ActionRepositoryPort, FeatureRepositoryPort, RecommendRepositoryPort

3. quadra-recommend-adapter-out 模块：
   - ActionRepositoryImpl（行为日志批量写入）
   - FeatureRepositoryImpl（特征表 upsert，含 AI embedding JSON 预留）
   - 定时任务调度器（Scheduler，放在 adapter-out/scheduler）

4. quadra-recommend-adapter-in-web 模块：
   - RecommendController（POST /api/v1/recommends/actions 等）
   - Scheduler 入口

【强约束】
- 不允许跨层调用
- 不允许出现 DAO 直连 Controller
- Repository 必须操作聚合根
- Domain 层不能出现 Spring 注解
- 定时任务逻辑放在 Adapter-out/scheduler，而非 Domain 层

请严格按照分层结构输出代码。
