请基于六边形架构，为 Quadra System 服务实现一个完整的用例：

【用例名称】
AdminLogin（管理员登录）

【业务要求】
- 使用用户名密码登录
- 密码使用 BCrypt 校验（在 application 层处理）
- 校验管理员状态（启用/禁用状态机）
- 生成管理端 JWT Token（与用户端隔离 Scope）
- 记录登录审计日志

【技术要求】
- JDK 21
- Spring Boot 3
- MyBatis-Plus
- MySQL 8

【必须输出以下内容】
1. quadra-system-domain 模块：
   - SysAdmin 聚合根（包含 create 工厂方法，状态机 enable/disable）
   - SysRole / SysMenu 聚合根
   - AdminStatus 枚举（ENABLED/DISABLED）
   - AdminCreatedEvent

2. quadra-system-application 模块：
   - AdminLoginCommand
   - AdminLoginUseCase 接口
   - SystemApplicationService 实现（包含 BCrypt 校验、JWT 生成）
   - AssignRoleToAdminUseCase、GrantMenuToRoleUseCase、GenerateDailyAnalysisUseCase
   - port.out：AdminRepositoryPort, RoleRepositoryPort, MenuRepositoryPort, EventPublisherPort, AuditLogPort

3. quadra-system-adapter-out 模块：
   - AdminRepositoryImpl（管理员 + 角色关联表的聚合持久化）
   - OutboxEventRepositoryImpl（写事件表，与业务同事务）
   - AuditLogAspect（审计日志 AOP 采集）

4. quadra-system-adapter-in-web 模块：
   - SystemController（POST /api/v1/system/admin/login 等）
   - Scheduler（日报聚合定时任务）

【强约束】
- 不允许跨层调用
- 不允许出现 DAO 直连 Controller
- Repository 必须操作聚合根
- Domain 层不能出现 Spring 注解
- 权限校验逻辑必须在拦截器/AOP 中，禁止在 Controller 中实现

请严格按照分层结构输出代码。
